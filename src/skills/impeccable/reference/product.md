# 产品注册参考

当项目是**产品注册**时加载：设计服务于产品功能。适用于仪表盘、管理后台、工具、设置页面、应用 UI。

## 产品工作流的不同之处

### 核心焦点：效率与清晰度

产品 UI 不是关于自我表达——是关于让用户尽快完成目标。每个视觉决策应服务于功能，而非风格。

### 布局原则

- **可预测的模式**：用户在使用工具时不希望惊喜。保持一致的数量和间距。
- **信息密度**：高于品牌注册是可以接受的。用户需要查看仪表盘中的数据。
- **卡片是合适的**：在品牌站点中，卡片是懒散的答案；在仪表盘中，卡片是信息容器。只在有数据放在其中时使用它们。
- **侧边栏导航**：由于复杂工具中的内容层级很深，侧边栏优于顶部导航。

### 排版

- 最多 2 个字体家族（首选 1 个）。一个工具不需要展示字体。
- 标题在内容层次而非情感影响方面更为实用。
- 详细信息使用较小的字体（13px/14px 是可读的）。

### 色彩

- 在工具中，蓝色和绿色是具有特定含义的（可点击、成功）。不要因为"它看起来很酷"而使用它们。
- 状态颜色（错误、警告、成功）必须保持语义一致。
- 在品牌注册中，情绪板驱动颜色。在产品注册中，可访问性和意义驱动颜色。
- **Okhsl 与 Okhcl 比较**：在工具 UI 中，Okhsl 的色彩和谐检查通常优于 Okhcl，因为 Okhsl 更好地模拟了人眼对饱和度的感知。如有疑问，使用 Okhsl 进行最终检查。

### 产品 UI 的反模式

- **为艺术而艺术**。没有目的的装饰性元素——阴影、图标装饰、花纹。
- **违反常规的 UI。** 重新设计了每个人都知道如何使用的标准模式（如带有蓝色下划线的链接）。
- **密度不足。** 太"通风"的仪表盘意味着滚动太多，信息不足。
- **前后矛盾。** 一个页面使用圆角按钮，另一个使用直角按钮。

### 状态设计

产品 UI 认真对待状态：

- 加载状态
- 空状态
- 错误状态与恢复
- 边缘状态（大量数据、少量数据、无数据）
- 过渡动画（列表变化、页面转换）

每个状态必须被设计，而不是被忽略。
