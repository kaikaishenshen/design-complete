# Layout 流程

修复间距问题、节奏和视觉层次。

## 何时使用

用户在说：「布局感觉不对」、「修复间距」或「东西感觉拥挤」。

## 怎么做

### 1. 间距审计
- 元素之间的间距是否一致？
- 各区域是否感觉拥挤或空洞？
- 遵循节奏系统（4px 基础 → 8、16、24、32、48、64、96）

### 2. 视觉层次
- 最重要的元素是否最突出？
- 次要元素是否退居次位？
- 阅读流程是否从左到右、从上到下工作？

### 3. 网格结构
- 一维排列使用 flexbox，二维使用 grid
- 无断点的响应式网格：`repeat(auto-fit, minmax(280px, 1fr))`
- 变化卡片/元素大小——不要所有东西都一样大

### 4. 反模式修复
- 移除嵌套卡片
- 展开过于狭窄的容器间距
- 检查并修复溢出文本
- 建立语义化 z-index 层级

### 5. 响应式
- 移动端：单列布局，堆叠内容
- 平板端：可选 2 列
- 桌面端：全宽网格，展现原本设计
