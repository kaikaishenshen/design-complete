# Clarify 流程

改进 UX 文案：使标签、错误消息、空状态和用户面向的文本更清晰、更具体、更有帮助。

## 何时使用

用户在说：「文案不清楚」或「修复错误消息」。

## 怎么做

### 按钮标签
- 使用动词 + 对象：「保存变更」而非「确定」
- 说明会发生什么：「删除项目」而非「是」
- 避免通用标签

### 错误消息
- 说明出了什么问题
- 说明如何修复它
- 对普通用户可理解，而非技术术语

### 空状态
- 说明为什么那里是空的
- 如果有，说明如何填充它
- 使用友好语气，但保持简短

### 一般 UX 文案
- 避免营销套话（由 AI 产生）
- 每个词都有其位置
- 不要重述标题
- 使用用户的语言，而非公司的语言

### 链接文本
- 独立成义：「查看定价方案」而非「点击这里」
- 屏幕阅读器会脱离上下文播报链接
