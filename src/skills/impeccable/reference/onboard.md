# Onboard 流程

设计首次使用流程、空状态和激活路径。帮助新用户从一开始就成功。

## 何时使用

用户在说：「设计引导流程」、「添加空状态」或「修复首次使用体验」。

## 怎么做

### 首次使用流程
- 分步引导而非一次性全部倾倒
- 使用渐进式披露：先问最重要的信息，后问细节
- 提供跳过选项——不是每个人都想被引导
- 显示进度（步骤 1/4）

### 空状态
- 解释为什么那里是空的
- 说明用户如何填充它
- 添加一个 CTA 开始使用
- 使用轻量级插图

### 激活
- 识别让用户成功的「巅峰时刻」
- 减少到达那个时刻的摩擦
- 不要求注册即可探索

### 原则
- 用户不想阅读——他们想做。展示，而非告知。
- 每个屏幕一个操作
- 衡量引导是否有效（完成率）
