# Colorize 流程

为单色或平淡的 UI 添加战略性色彩。色彩应该感觉*有意*而不是*装饰性*。

## 何时使用

用户在说："颜色太平淡了。"或者"给它增添色彩。"

## 步骤 1：建立锚点

如果还没有品牌色，运行 `palette.mjs` 获取种子色。否则，使用现有的品牌色作为锚点。

## 步骤 2：识别色彩角色

为颜色分配特定任务，而不是喷洒它：

1. **强调色**：用于 CTA、链接、交互元素。有限地使用。
2. **表面色**：用于卡片、侧边栏、区域分隔。从品牌色调的有色中性色中选择。
3. **背景色**：整体画布。中性或轻微染色。
4. **状态色**：成功、错误、警告、信息。如果是产品 UI，这些是必需的。

## 步骤 3：战略性地应用

- **20% 规则。** 你的界面应该大约 80% 中性色，20% 色彩。
- **在关键点上使用色彩。** 导航、CTA、活动状态——而非装饰性标题。
- **色彩应传达意义。** 在产品的待办事项中，红色用于逾期，绿色用于完成，蓝色用于进行中。
- **测试对比度。** 每个颜色使用必须在浅色背景下测试对比度。正文与背景的对比度 ≥ 4.5:1。

## 步骤 4：为色彩调整设计

- 提交调色板作为 token（CSS 自定义属性或设计 token 文件）
- 使调色板可以被未来的命令调整
