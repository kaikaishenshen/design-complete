# Audit 流程

对现有代码或无头浏览器目标运行技术质量检查。检测可访问性问题、性能风险、响应式问题和反模式。

## 步骤 1：确定目标

目标可以是 HTML 文件、CSS 文件、一个目录或者一个 URL。如果未提供，从上下文推断。

## 步骤 2：运行检测器

```bash
node .workbuddy/skills/impeccable/scripts/detect.mjs --json [target]
```

收集所有命中结果。按严重程度分类：

### 严重（P0）
- 对比度违规（文本 < 4.5:1）
- 嵌套卡片
- 溢出文本
- 键盘陷阱或缺失焦点指示器
- 缺失 ARIA 标签或角色

### 警告（P1）
- 过窄的内边距（间距 < 8px 在容器中）
- 缺少 `lang` 属性在 `<html>` 上
- 跳过标题层级（h1→h3）
- 缺少 `alt` 文本在图片上
- 触摸目标 < 44px（移动端）

### 信息（P2）
- AI 模板迹象（过度使用的字体、侧边条纹、眼眉标）
- 默认字体（Inter、Roboto）
- 布局使用纯灰阴影
- 动画使用弹跳缓动

## 步骤 3：无头浏览器检查（可选）

如果无头浏览器可用，在视口中加载页面并检查：

- 加载行为：是否有布局偏移？
- 字体加载：文本是否在字体加载时闪烁（FOUT）？
- 滚动行为：平滑还是卡顿？
- 视口适配：页面是否适应设备尺寸？

## 步骤 4：报告

以结构化格式呈现发现：
1. **严重问题**：必须修复
2. **警告**：应修复
3. **建议**：考虑修复

每个发现应包括：
- 文件/位置
- 当前值
- 预期值
- 修复建议

## 范围

Audit 检查：
- ✅ HTML 语义/结构
- ✅ CSS（对比度、间距、溢出）
- ✅ 可访问性（aria、焦点、语言、alt）
- ✅ 响应式（meta viewport、断点）
- ✅ 反模式（AI 模板）
- ❌ JavaScript 逻辑/功能
- ❌ 后端性能
- ❌ 图片优化（为单独工具保留）
