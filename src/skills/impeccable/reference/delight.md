# Delight 流程

添加个性和令人难忘的触感。令人愉悦的细节让一个界面从「好」变成「令人难忘」。

## 何时使用

用户在说：「添加一些快乐」或「让它令人难忘」。

## 怎么做

- **微交互**：切换开关动画、成功打勾动画、数字变化的计数器动画
- **空状态**：使用友好插图而非枯燥的「无数据」消息
- **加载状态**：骨架屏而非旋转器
- **表单反馈**：内联验证、成功动画、有趣的错误恢复
- **页面过渡**：微妙的路由过渡或滚动触发的部分入场
- **悬停彩蛋**：卡片翻转、图标变化、有趣的指针效果

## 原则

- 令人愉悦不应妨碍可用性
- 快乐应该让人微笑，而不是等待
- 使用动效时，保持 ≤ 300ms。更多会感觉迟滞
- 如果用户看不到它（折叠下方，隐藏标签中），不要在那里放快乐
