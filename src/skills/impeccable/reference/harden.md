# Harden 流程

为生产环境做好准备：添加错误处理、国际化支持、文本溢出处理和边界情况。

## 何时使用

用户在说：「加固它」或「为上线做好准备」。

## 怎么做

### 错误处理
- 为每个用户操作添加错误状态
- API 错误应有用户友好的消息
- 网络故障应有重试机制
- 表单应有内联验证

### 文本溢出
- 在狭窄容器中使用 `text-overflow: ellipsis`
- 长词使用 `overflow-wrap: break-word`
- 在移动端测试标题

### 国际化
- 提取 UI 字符串到资源文件中
- 支持 RTL 语言布局
- 数字和日期格式应基于区域

### 边界情况
- 极端数据量（一个项目 vs 一万个项目）
- 空状态（没有数据、初始状态）
- 非常长的文本（名称、描述）
- 特殊字符（HTML 实体、emoji、Unicode）
- 键盘导航（Tab 键顺序、Enter 键提交）

### 性能
- 大列表使用代码分割和懒加载
- 图片懒加载
- 添加骨架屏加载状态
