# Document 流程

从现有项目代码生成 DESIGN.md。当项目已有代码但无设计文档时使用。

## 步骤 1：扫描

通过检查现有文件查找设计模式：

- CSS 文件中的颜色值
- CSS 自定义属性（`:root` 变量）
- 字体导入
- 组件结构
- 间距模式

## 步骤 2：提取

提取到设计系统：

### 提取内容
- **排版**：使用中的字体、字号、行高、字重
- **颜色**：品牌色、背景色、强调色、文字色
- **间距**：常用间距值、边距模式
- **组件**：重复使用的 UI 元素

## 步骤 3：写 DESIGN.md

在项目根目录创建 `DESIGN.md`，包含：

```markdown
# 设计系统

## 品牌概述
[一句话描述产品]

## 排版
- 展示字体：[字体名]
- 正文字体：[字体名]
- 比例：[比例因子]

## 调色板
- 主色：[OKLCH 值]
- 背景色：[OKLCH 值]  
- 表面色：[OKLCH 值]
- 强调色：[OKLCH 值]
- 文字色：[OKLCH 值]

## 间距
- 基准单位：[值]
- 常用间距：[列表]
```

## 步骤 4：呈现

向用户展示生成的 DESIGN.md 让他们确认。他们可能想添加或调整内容。
