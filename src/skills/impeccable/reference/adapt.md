# Adapt 流程

适配不同设备和屏幕尺寸。确保设计在所有视口上正常工作。

## 何时使用

用户在说：「适配移动端」或「在平板设备上看起来不对」。

## 怎么做

### 移动端（< 640px）
- 单列布局
- 堆叠内容——最重要的在最上面
- 触摸目标 ≥ 44px
- 链接和按钮之间间距充足
- 考虑底部导航栏替代顶部导航
- 字体不小于 16px（防止 iOS 缩放）

### 平板端（640–1024px）
- 可选 2 列布局
- 侧边栏可折叠
- 预览网格：`repeat(auto-fit, minmax(280px, 1fr))`

### 桌面端（> 1024px）
- 展现完整设计意图
- 多列布局
- 完整的侧边栏

### 通用规则
- 使用 container queries 匹配组件上下文
- 不要隐藏内容——重新排列它
- 文本永远不应溢出其容器
- 测试每个断点的实际设备宽度
